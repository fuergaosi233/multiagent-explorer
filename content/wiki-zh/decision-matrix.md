# 决策矩阵

## 按任务特征

| 任务特征 | 推荐 | 避免 | 原因 |
|---|---|---|---|
| 单轮专家查询 | 智能体即工具 | 交接 / 群聊 | 宿主可以简单保持控制 |
| 多轮领域接管 | 交接 | 智能体即工具 | 专家需要直接与用户对话 |
| 固定流程 | 顺序管道 / 工作流 | 群体 | 确定性是关键 |
| 多个独立子任务 | 并行扇出 / 收集 | 顺序 | 并行性带来最大加速 |
| 长期、复杂任务 | 图工作流 + 任务注册表 | 纯提示编排 | 需要恢复、取消、追踪 |
| 需要质量审查 | 生成器-批评者 / 精炼循环 | 单智能体自我评分 | 自我评分经常重复相同错误 |
| 比较多个选项 | 辩论 / 投票 / MoA | 单一路径 | 多候选揭示冲突 |
| 异步多方工作 | 黑板 / 事件总线 | 群聊 | 共享状态比长对话更稳定 |
| 高风险操作 | 人在回路 + 防护栏 | 完全自动化 | 权限和责任必须明确 |
| 全代码库审计 | 动态工作流 + Parallel + Verifier + Worktree | 单智能体 | 按文件扇出 + 大规模对抗审查 |
| 500 文件迁移 | 动态工作流 + Worktree 隔离 + 测试循环 | 单智能体 / 群聊 | 并行按文件移植，build/test 修复循环，PR 输出 |
| 交叉验证研究 | 动态工作流 + Deep Research 式验证 | 单智能体 | 综合前对每条 claim 独立验证 |
| 需要中途人工签字 | 拆为多个工作流 + 人在回路 gate | 单一长工作流 | 工作流不接受中途用户输入 |
| 跨工具生态 | MCP | 自定义工具协议 | 降低集成成本 |
| 跨智能体 / 跨供应商 | A2A / ACP | 私有 RPC | 更适合互操作性 |
| IDE 到编码智能体 | 智能体客户端协议 | 临时 HTTP API | 更接近编码智能体标准入门 |

## 按工程阶段

| 阶段 | 目标 | 推荐栈 |
|---|---|---|
| 演示 | 证明可行 | 单智能体 + 工具 + 智能体即工具 |
| MVP | 可用 | 监管者 + 交接 + 追踪 |
| 内部平台 | 稳定、可恢复 | 图工作流 + 任务注册表 + 黑板 + MCP |
| 生产平台 | 安全、可审计 | 运行时 + 防护栏 + 人在回路 + 事件总线 + 工作空间隔离 |
| 生态 | 连接到其他系统 | MCP + A2A/ACP + 智能体客户端协议 |

## 反模式

1. **首先选择群聊。** 看起来像多智能体，但很少收敛。
2. **没有状态机的长期任务。** 半完成的任务无法恢复或调试。
3. **所有智能体共享一个上下文。** 严重污染，权限边界消失。
4. **没有验证器的代码生成。** 智能体自信地产生错误的 diff。
5. **没有事件日志。** 出问题时无法重建发生了什么。
