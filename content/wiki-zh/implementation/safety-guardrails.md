# 安全、权限与护栏

多智能体平台最大的风险不是"错误答案" —— 而是当多个智能体触及工具、文件系统、命令行、网络和外部系统时，出现的不可控动作链。

## 权限层级

| 级别 | 动作 | 默认策略 |
|---|---|---|
| L0 | 只读上下文、只读文档 | 允许 |
| L1 | 文件读取、搜索、静态分析 | 允许，但记录日志 |
| L2 | 文件写入、配置更改 | 需要策略检查 |
| L3 | 命令行、依赖安装、网络调用 | 需要审批或沙箱 |
| L4 | git 提交、部署、数据库写入 | 需要人工审批 |
| L5 | 生产环境、财务、权限更改 | 默认拒绝或严格审批 |

## 护栏类型

| 类型 | 示例 |
|---|---|
| 输入护栏 | 检测提示注入、超出范围请求 |
| 工具护栏 | 限制命令行命令、网络域、文件路径 |
| 输出护栏 | 检测信息泄漏、危险建议、策略违规 |
| 预算护栏 | 限制令牌数、时间、并发数 |
| 状态护栏 | 防止黑板污染 |
| 交接护栏 | 防止循环和未授权接管 |

## 命令行策略示例

```ts
const shellPolicy = {
  allow: ["ls", "cat", "grep", "rg", "sed", "python -m pytest"],
  deny: ["rm -rf", "sudo", "curl | sh", "chmod 777", "docker system prune"],
  requireApproval: ["git push", "npm publish", "kubectl", "terraform apply"],
};
```

## 审批卡片必须包含

- `action` —— 将要执行什么
- `reason` —— 为什么执行
- `scope` —— 影响范围
- `diff` —— 将会改变什么
- `rollback` —— 如何撤销
- `risk` —— 严重程度层级
- `timeout` —— 审批超时后的处理方式

## 常见失败

1. 子智能体调用了主智能体不知道的工具。
2. 交接后权限扩大。
3. MCP 服务器暴露了过多工具。
4. 审批卡片只显示一行文字，没有差异对比或影响说明。
5. 追踪记录了敏感信息但未脱敏。
